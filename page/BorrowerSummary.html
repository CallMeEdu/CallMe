Borrower Summary

THE PROBLEM

The loan application is cumbersome

The loan application can be a cumbersome process for borrowers, requiring lot of manual entry. Requesting data that is already stored in other systems is repetitive and can lead to discrepancies.

The sections borrowers have to fill out in the loan application

+ Why is it such a big deal?

Lead generation is a critical part of Roostify's business: majority of our clients use Roostify as a lead generation tool. The ability for the borrower to quickly submit a loan application is critical to the success of the business.

THE OBJECTIVE

Speed up loan application with data prefill

We can further help borrowers complete loans applications by collecting data from external data sources automatically. With an easier application experience, we can lower the drop off rate and lenders are more likely to generate more leads.

Product Requirements

Key User Stories

The prefill service is expected to have great impact on borrower experience by connecting multiple data sources and greatly automating the loan application process. But it can't be built within one day. We break down the product requirements through feasible phases in alignment with our clients' expectations.

As a borrrower, when I start a new loan application and data is pre-filled on my behalf (e.g, with data known from my bank), I want to see a summary of all pre-filled data so that I'm not caught by surprise and have a moment of delight.

MVP
Borrower Experience

As a loan officer, I would like to know where pre-filled data has originated from so that I know whether it is a trusted source and the level of validation needed.

Phase 2

Lender Experience

As a bank admin, when creating a complete prefill solution for applicants, I want to link many data sources, so that Roostify can prefill data from different systems

Phase 3

Multiple Data Sources

The Goals

Based on the problem space and requirements, we set up the following goals:

Trust

Inspire trust by ensuring the data accuracy and transparency

Scalability

Build scalability with new data sources in the future

Design Walkthrough

Here is a Scenario

Digital Lender (Our fictitious lender) uses Roostify for digital loan applications.

Neha is an existing account holder with Digital Lender and decides she wants to apply for a mortgage loan with Digital Lender.

Neha gives consent to prefill her loan appplication

Neha logs in through Digital Lender SSO. Digital Lender already has some of her information in the system. She likes the transparency and reassurance, and feels more in control of the process.

Prefill progress

She is able to see the prefill progress in real time and understand how much time left.

Loan Application Summary

She is directed to the loan application summary page where. She quickly understood what has been filled out and what's remaining.

Field Indicator

She quickly locates the prefilled fields to review/edit
